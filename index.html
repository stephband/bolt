Bolt is a front-end toolkit of CSS classes and touch-ready evented behaviours. It was born at Urturn and grew up at Cruncher. Bolt requires jQuery but is otherwise agnostic. It has been put into production on top of both Rails and Django, and integrated with Backbone and various other app frameworks.

CSS

Bolt is designed to be flexibly sized and responsive, and to progressively enhance. Base classes concentrate on layout techniques, and optional vertical rhythms and horizontal grids are included. Bolt does not give you any colour. Colours change between projects, where layout techniques generally do not.

Bolt prefers CSS over JavaScript. It uses class changes and CSS transitions rather than JS animations.

JS

Bolt's classes come with behaviour attached. Bolt uses jQuery special events to provide default behaviour for dropdowns, dialogs, tabs, slides, tips and anything else that has visible/hidden states.

Events are delegated, so there are no plugins to call to make these interactive elements work. Creating a tooltip, for example, is as simple as writing: 

<a href="#time">Time is an illusion.</a>

<p class="tip" id="time">Lunchtime, doubly so.</p>
